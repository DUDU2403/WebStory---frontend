# 🔍 Guia de Diagnóstico - Erro de Credenciais ao Logar na Loja\n\n## **O problema que você está enfrentando**\n\nVocê criou uma loja ou tentou logar, mas recebe:\n```\n❌ \"Credenciais inválidas\" ou \"E-mail ou senha inválidos\"\n```\n\nIsso pode ter várias causas. Vamos investigar:\n\n---\n\n## **Checklist de Diagnóstico**\n\n### **1️⃣ Primeira coisa: Há uma loja criada?**\n\nVá para o **Painel Admin** (`/admin`):\n- Login com email do admin\n- Clique em **\"🏪 Lojas\"**\n- **Há alguma loja listada?**\n\n**Se NÃO há lojas:**\n→ Você precisa criar uma loja primeiro. Veja a seção \"Como criar uma loja\" abaixo.\n\n**Se SIM, há lojas:**\n→ Vá para o próximo passo.\n\n---\n\n### **2️⃣ A loja está ATIVA?**\n\nNo admin, na aba **\"🏪 Lojas\"**:\n- Procure a loja que você quer acessar\n- Veja se há um toggle ou status mostrando **\"Ativa\"** ✅ ou **\"Inativa\"** ❌\n\n**Se estiver INATIVA:**\n→ Clique no toggle para ativar\n→ Tente logar novamente\n\n---\n\n### **3️⃣ Você tem a CHAVE DE ACESSO certa?**\n\n**Se você AINDA NÃO criou uma loja:**\n\nNo Admin Panel:\n1. Clique em **\"🔑 Chaves\"**\n2. Clique em **\"+ Gerar nova chave\"**\n3. Copia a chave que aparece (algo como `AK-A1B2C3D4`)\n\nDepois, vá para **\"Criar loja\"** em Login Vendedor:\n1. Preencha: Nome, Email, Telefone, Senha\n2. Clique em **\"Próximo\"**\n3. Cole a chave copiada\n4. Clique em **\"Criar loja\"**\n\nDepois disso, você consegue fazer login.\n\n---\n\n## **Como Criar uma Loja (Passo a Passo)**\n\n### **Pré-requisito: Ter uma chave de acesso**\n\n1. **Acesse o Admin Panel** → `/admin`\n   - Email: `admin@webstory.com` (ou o email configurado)\n   - Senha: a que você definou no primeiro acesso\n\n2. **Gere uma chave:**\n   - Clique em **\"🔑 Chaves\"** no header\n   - Clique em **\"+ Gerar nova chave\"**\n   - Uma chave `AK-XXXXXXXX` será gerada e copiada automaticamente\n\n3. **Vá para Login Vendedor** → `/login-vendedor`\n   - Clique em **\"Criar loja\"**\n   - Preencha o formulário (Etapa 1):\n     - Nome da loja\n     - E-mail (use um email diferente do admin)\n     - Telefone/WhatsApp\n     - Senha\n   - Clique em **\"Próximo\"**\n\n4. **Etapa 2 - Cole a chave:**\n   - Cole a chave `AK-XXXXXXXX` no campo \"Chave de acesso\"\n   - (Opcional) CNPJ e Endereço\n   - Clique em **\"Criar loja\"**\n\n5. **Sucesso! Agora você pode logar:**\n   - Email: o que você usou no cadastro\n   - Senha: a que você definou\n   - Tipo: **\"Dono\"** (ou \"Funcionário\" se for um funcionário)\n\n---\n\n## **❌ Erros Comuns e Soluções**\n\n### **Erro 1: \"Chave inválida\" ao criar loja**\n- **Causa:** A chave não existe no banco ou já foi usada\n- **Solução:** Gere uma nova chave no admin\n\n### **Erro 2: \"Chave já utilizada\" ao criar loja**\n- **Causa:** Essa chave foi usada para criar outra loja\n- **Solução:** Gere uma nova chave no admin (agora você pode deletar chaves usadas também)\n\n### **Erro 3: \"Loja desativada\" ao tentar logar**\n- **Causa:** A loja existe mas está marcada como inativa\n- **Solução:** No admin, vá em \"🏪 Lojas\" e ative a loja\n\n### **Erro 4: \"E-mail já cadastrado\" ao criar loja**\n- **Causa:** Esse email já foi usado para outra loja\n- **Solução:** Use um email diferente\n\n### **Erro 5: \"E-mail ou senha inválidos\" ao tentar logar**\n- **Causa:** Email ou senha estão errados\n- **Solução:** \n  - Verifique se digitou corretamente\n  - Se perdeu a senha, exclua a loja no admin e crie uma nova\n\n---\n\n## **Testar o Fluxo Completo**\n\nUma vez que a loja está criada e logada, você pode:\n\n1. **Cadastrar produtos:**\n   - Dashboard → \"Produtos\" (ou similar)\n   - Clicar em \"+ Novo Produto\"\n   - Preencher dados\n\n2. **Clientes registram e fazem compras:**\n   - Ir para `/auth-cliente` ou clicar em \"Sou cliente\"\n   - Registrar com email diferente do dono\n   - Fazer login como cliente\n   - Ver e comprar produtos\n\n---\n\n## **Se Ainda Não Funcionar**\n\n### **Opção 1: Verificar Console do Navegador**\n1. Abra o navegador (F12)\n2. Clique em **\"Console\"** ou **\"Network\"**\n3. Tente fazer login novamente\n4. Veja qual é a resposta exata do servidor\n\n### **Opção 2: Verificar no Terminal**\nSe tiver acesso ao MongoDB:\n\n```bash\n# Conectar ao MongoDB\nmongosh \"sua_connection_string\"\n\n# Ver se há lojas\ndb.lojas.find({})\n\n# Ver se há chaves não usadas\ndb.chavedacesso.find({ usada: false })\n\n# Ver dados de uma loja específica (sem senha)\ndb.lojas.find({ email: \"seu@email.com\" }, { senha: 0 })\n```\n\n### **Opção 3: Reiniciar tudo**\n1. Limpar browser cache (Ctrl+Shift+Delete)\n2. Fechar e reabrir o navegador\n3. Tentar novamente\n\n---\n\n## **⚠️ Importante: Permissões e Status da Loja**\n\nAgora foi corrigido no código:\n- ✅ Clientes **PODEM registrar** mesmo se a loja estiver inativa\n- ✅ Produtos **APARECEM** mesmo se a loja estiver inativa\n- ✅ Admin **PODE deletar chaves** mesmo se já foram usadas\n\nMas para **FAZER LOGIN** como dono/funcionário:\n- ❌ Precisa que a loja exista no banco\n- ✅ Pode estar inativa (mas verá mensagem \"Loja desativada\")\n\n---\n\n## **Resumo Rápido do Fluxo**\n\n```\n┌─ ADMIN PANEL ─────────────┐\n│ Login → Gerar Chave       │\n└───────────────────────────┘\n           ↓\n┌─ CRIAR LOJA ──────────────┐\n│ Login Vendedor            │\n│ → Criar loja              │\n│ → Cole a chave            │\n└───────────────────────────┘\n           ↓\n┌─ LOGAR COMO DONO ─────────┐\n│ Login Vendedor            │\n│ → Email e senha           │\n│ → Dashboard               │\n└───────────────────────────┘\n           ↓\n┌─ CLIENTE REGISTRA ────────┐\n│ Auth Cliente              │\n│ → Registrar               │\n│ → Logar                   │\n│ → Ver produtos            │\n└───────────────────────────┘\n```\n\n---\n\n## **Dúvidas?**\n\nVerifique o arquivo `FIXES_APPLIED.md` para mais detalhes técnicos sobre as correções aplicadas.\n"
